# 安全说明

## 运行边界

- 代码通过列表参数调用 `lark-cli`，不使用 `shell=True`。
- 身份认证由 `lark-cli auth login` 管理；本代码不读取、保存或上传认证密钥。
- 真实飞书资源标识、章节内容、图片、处理 JSON、缓存、快照和验证配置只允许存放在私有运行工作区。
- 大纲与云端状态分别存放在 `config/outline.json` 和 `state/remote_nodes.json`；Token 不得进入大纲。
- 章节产物必须通过 `chapter_id` 或精确标题与目标节点绑定；仅序号相同时拒绝写入。
- 覆写云端页面前，脚本在私有工作区创建受限权限快照；白板重绘失败时尝试回滚。

## 发布控制

发布包必须通过 `python3 tools/build_release.py` 生成。该流程使用允许清单，只复制 Skill 所需的代码、说明和合成示例，并对最终暂存包运行敏感数据检查。

禁止直接执行以下操作作为发布手段：

- 从仓库根目录创建 ZIP
- 上传整个仓库目录到 Skill 市场
- 使用 `git add -A` 后未经审查地提交

`.gitignore` 不等于发布边界：它不能清除 Git 历史，也不能排除手动压缩时包含的文件。

## 报告问题

请勿在 Issue、日志或聊天中粘贴真实 `node_token`、`obj_token`、空间标识、文档 XML、课程原文或认证材料。发现泄露时先收紧资源分享权限，再停止发布、保留取证并清理历史。
