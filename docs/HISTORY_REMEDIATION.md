# 公开 Git 历史处置方案

目录分层和发布白名单只能防止未来泄露，不能删除已经推送的 Git blob。

## 当前边界

`tools/clean_history_plan.sh` 只会创建一个本地清理副本：

```bash
bash tools/clean_history_plan.sh
```

它不会修改 `origin`、不会 force-push、不会删除原仓库。必须先审查本地副本，再决定是否创建新的公开仓库或改写既有远端历史。

## 建议顺序

1. 将现有公开仓库临时设为私有或暂停继续发布。
2. 复核曾出现在历史中的飞书知识库、页面和分享链接权限；必要时迁移或重建资源。
3. 在本地运行清理脚本，检查清理后的提交历史和最终发布 ZIP。
4. 选择其一：新建干净公开仓库（推荐）或在团队确认后重写现有远端历史并 force-push。
5. 通知协作者、fork 持有者和已克隆者：旧 clone/缓存不能由历史重写自动删除。
6. 若需要平台级缓存/搜索清除，按 GitHub 的敏感数据删除流程提交申请。

## 不能承诺的事

历史重写不能撤回已经被下载、镜像、搜索索引或缓存的数据。因此请将已出现的资源标识按已暴露信息处理，而不是只依赖 Git 清理。
